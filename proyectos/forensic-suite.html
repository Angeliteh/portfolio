The Output

The system outputs a formal PDF document containing an inventory of all extracted files. Beside each file is its computed SHA-256 hash. Because the hash is calculated at the exact moment the buffer writes to disk, it is mathematically impossible to alter the extracted media without the hash changing, thus preserving the chain of custody for court.

The Problem

Screenshots don't hold up in court. Chain of custody breaks easily.

In civil and labor litigation, WhatsApp messages are increasingly critical evidence. But screenshots are trivially manipulable — any attorney can argue the image was edited in Photoshop or via browser DevTools. Courts routinely reject them.

Enterprise-grade forensic tools (Cellebrite, Magnet AXIOM) cost thousands of dollars annually and require specialized certifications. Independent attorneys, private investigators, and mid-scale forensic analysts had no affordable alternative that met legal admissibility standards.

System Architecture
Electron Main  (main.js)
    └─ IPC Bridge ↔ Renderer UI (HTML/CSS/JS)
    └─ Orchestrates Logical Module (WhatsApp)
    └─ Orchestrates Physical Module (ADB)
    └─ Fixed window sizing → preserves visual consistency

Logical Module  (fase3_extractor_real.js)
    └─ Playwright/Chromium: headless:false (user sees QR)
    └─ Infinite scroll automation → extracts full chat history
    └─ Reads data-pre-plain-text metadata (real author + timestamp)
    └─ Writes CSV → immediately calculates SHA-256 hash

Physical Module  (3_extraccion_galeria_adb.js)
    └─ Spawns adb.exe from local bin/ (zero external dependency)
    └─ USB or WiFi pairing via dynamic PIN injection to stdin
    └─ Recursive pull of DCIM, WhatsApp media, Downloads
    └─ Per-file SHA-256 hash calculated on every extracted item

Report Engine  (PDFKit)
    └─ Auto-generates forensic PDF certificate
    └─ Contains: UTC timestamp, analyst name, file inventory
    └─ SHA-256 hash per item in Courier monospace
    └─ Legal clause: any modification invalidates all hashes
Forensic Integrity System
Immediate SHA-256 Hashing

Hash is calculated at the buffer level the moment data is written to disk — before the PDF is generated. Any post-extraction modification changes the hash, proving tampering mathematically.

Automated Forensic Certificate (PDFKit)

PDF contains: extraction date/time in ISO 8601, software version, analyst name, module used, and SHA-256 fingerprint for every extracted file. Designed for direct submission to a judge or notary.

Dual Extraction: Web + Physical

Logical: Playwright scrolls WhatsApp Web and reads native DOM metadata (author phone number, message timestamp). Physical: ADB extracts EXIF metadata from images, including GPS coordinates and capture date.

100% Local — Air-Gap Capable

No evidence data is transmitted over the internet. ADB binary is bundled in bin/ — no Android SDK installation required. Runs in courtrooms or locations without connectivity.

Technical Challenges & Solutions

Inadmissibility of Screenshots

Problem

Lawyers constantly lose civil cases because opposing counsel argues that screenshots of WhatsApp are easily forged via Photoshop or browser DevTools.

Solution

Zero-time gap cryptography. The system calculates the SHA-256 hash the precise millisecond the buffer is written to disk, guaranteeing mathematical immutability.

Hardware Dependencies & Portability

Problem

To perform physical extractions, typical forensic tools require installing heavy Android SDKs, making it impossible to run the tool on random courtroom laptops.

Solution

Electron bundling. The core adb.exe binary is shipped inside the app's internal bin folder. The app acts as a zero-dependency portable executable.
