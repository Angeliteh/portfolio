El resultado

El sistema genera un PDF formal con inventario de todos los archivos extraídos y su hash SHA-256 calculado. Como ese hash se obtiene exactamente al escribir el buffer a disco, alterar la evidencia después cambia el hash y protege la cadena de custodia.

El problema

Las capturas no sostienen un juicio. La cadena de custodia se rompe fácil.

En litigios civiles y laborales, los mensajes de WhatsApp son evidencia crítica. Pero las capturas son fácilmente manipulables y cualquier abogado puede alegar edición con Photoshop o DevTools. Los tribunales suelen rechazarlas.

Las herramientas forenses enterprise cuestan miles de dólares al año y exigen certificaciones especializadas. Abogados independientes, peritos y agencias privadas no tenían una alternativa accesible con estándares de admisibilidad legal.

Arquitectura del sistema
Electron Main  (main.js)
    └─ IPC Bridge ↔ Renderer UI (HTML/CSS/JS)
    └─ Orchestrates Logical Module (WhatsApp)
    └─ Orchestrates Physical Module (ADB)
    └─ Fixed window sizing → preserves visual consistency

Logical Module  (fase3_extractor_real.js)
    └─ Playwright/Chromium: headless:false (user sees QR)
    └─ Infinite scroll automation → extracts full chat history
    └─ Reads data-pre-plain-text metadata (real author + timestamp)
    └─ Writes CSV → immediately calculates SHA-256 hash

Physical Module  (3_extraccion_galeria_adb.js)
    └─ Spawns adb.exe from local bin/ (zero external dependency)
    └─ USB or WiFi pairing via dynamic PIN injection to stdin
    └─ Recursive pull of DCIM, WhatsApp media, Downloads
    └─ Per-file SHA-256 hash calculated on every extracted item

Report Engine  (PDFKit)
    └─ Auto-generates forensic PDF certificate
    └─ Contains: UTC timestamp, analyst name, file inventory
    └─ SHA-256 hash per item in Courier monospace
    └─ Legal clause: any modification invalidates all hashes
Sistema de integridad forense
Hash SHA-256 inmediato

El hash se calcula a nivel buffer en el instante exacto en que los datos se escriben a disco, antes de generar el PDF. Cualquier modificación posterior cambia el hash y prueba alteración matemática.

Certificado forense automatizado

El PDF contiene fecha y hora ISO 8601, versión del software, nombre del analista, módulo utilizado y fingerprint SHA-256 por archivo extraído. Está diseñado para presentarse directamente ante juez o fedatario.

Extracción dual: web y física

Lógica: Playwright recorre WhatsApp Web y lee metadatos nativos del DOM como autor y timestamp. Física: ADB extrae metadatos EXIF de imágenes, incluidas coordenadas GPS y fecha de captura.

100% local, apto para air-gap

Ningún dato de evidencia se transmite por internet. El binario ADB va incluido en bin/, sin requerir Android SDK. Puede ejecutarse en salas de juicio o lugares sin conectividad.

Desafíos técnicos y soluciones

Inadmisibilidad de capturas

Problema

Abogados pierden casos porque la contraparte argumenta que las capturas de WhatsApp pueden falsificarse fácilmente con Photoshop o DevTools.

Solución

Criptografía con gap cero en el tiempo. El sistema calcula el hash SHA-256 en el milisegundo exacto en que el buffer se escribe a disco, garantizando inmutabilidad matemática.

Dependencias de hardware y portabilidad

Problema

Para hacer extracciones físicas, muchas herramientas exigen instalar Android SDKs pesados, imposibilitando ejecutarlas en laptops improvisadas de sala.

Solución

Empaquetado con Electron. El binario central adb.exe viaja dentro de la carpeta interna `bin`, permitiendo que la app funcione como ejecutable portable sin dependencias externas.
