System Overview

Hybrid production-ready system combining a conversational AI assistant with a traditional administrative interface. School principals query in natural language for searches, statistics, and official certificates — while administrators maintain full CRUD control. 100% local (SQLite), distributed as a compiled PyInstaller executable.

Demo Flow (mirrors the 18-min video)

Part A — Classic Management

  • Open Classic Management → show student records (name, CURP, matricula, grade, group, shift)
  • Transform card: select external PDF → extract data → generate correctly formatted certificate
  • Search card: find by name, grade, group, shift, matricula, CURP
  • Management card (CRUD): edit, validate, save with visual control

Part B — AI Chat (automated)

  • "Find students with last name García" → list results
  • Contextual follow-up: "From that list, only evening shift" → filtered
  • Progressive refinement: "Now third grade" → "Now group A"
  • "How many students are in third grade?" → count + distribution
  • "Generate school certificate for [Name]" → PDF opens automatically
  • Attach external PDF → "Transform this PDF to official certificate" → extracted + new PDF
The Problem

School secretaries spent 15–20 minutes per certificate. Manually.

In Mexican primary schools, generating a student certificate (constancia de estudios) requires locating the physical file, verifying CURP and matricula data, filling a formatted document, and printing — often with errors. For a school handling 200+ certificates per month, this consumed days of administrative work.

Existing school management software was either too expensive, too complex to deploy locally, or simply didn't include AI-powered natural language interaction — forcing staff to navigate complex menus for basic queries.

AI Architecture: Master → Student → Help

Intent-based routing with conversational memory.

Master Interpreter
   └─ Understands user intent + sub-intents from natural language
   └─ MasterPromptManager: maintains conversation_stack + awaiting state
   └─ Handles ambiguity: "él", "el segundo", "de esos..." → resolves via context
   └─ Routes to: StudentSpecialist | HelpSpecialist

Student Specialist
   └─ Maps natural language to SQL queries + filters
   └─ Service-Repository pattern: AI layer separate from DB layer
   └─ PyQt Signals/Slots: AI runs in background thread → no UI freeze
   └─ Outputs: student list, statistics, PDF trigger

Help Specialist
   └─ Explains system capabilities in plain language
   └─ Guides users through available features

Data & Distribution
   └─ SQLite (local, single source of truth)
   └─ wkhtmltopdf → PDF generation from HTML templates
   └─ pdfplumber → extract data from existing PDFs
   └─ PyInstaller: compiled exe with monkey-patch for google-generativeai
Key Features
Dual Interface — AI Chat + Classic Management

Two complementary views sharing the same SQLite backend. Chat for speed, CRUD for precision. Users switch between them without losing context.

Certificate Generation & Transformation

Generates official PDF certificates from templates in 1–5 seconds. Can also receive an external PDF, extract its data with pdfplumber, and reformat it into the official institution template.

Progressive Natural Language Filtering

Chain queries conversationally: "García" → "evening shift" → "third grade" → "group A". Each message refines the previous result without resetting context.

Zero-dependency Distribution

Ships as a single compiled executable via PyInstaller. Includes custom monkey-patch for google-generativeai compatibility, bundled wkhtmltopdf, and automatic .env loading.

Technical Challenges & Solutions

google-generativeai + PyInstaller

Problem

Import paths and runtime broken after freezing. .env variables inaccessible. AI library imports incompatible with PyInstaller's module collection.

Solution

Early monkey-patch before any AI imports. Manual .env loading within the compiled executable. Custom hooks + resource paths with multi-level model fallback.

Intent Routing + Conversational Context

Problem

Ambiguous references ("him", "the second one", "from those..."), deciding when to use context and how to resolve pronoun references across turns.

Solution

Unified analysis via MasterPromptManager. conversation_stack with "awaiting" states. Intelligent continuation logic that compares current vs. previous intent.

Data Consistency Across Modules

Problem

Name and field variants across manually entered records broke searches and broke PDF templates with incorrect field values.

Solution

Centralized normalization layer in Config class. Validation with user-visible warnings before save. Sort order configurable per-institution.
