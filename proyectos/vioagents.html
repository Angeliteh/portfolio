The Problem

Traditional chatbots are rigid. LLM wrappers hallucinate.

SMBs lose leads every day because they can't respond instantly outside business hours. Traditional rule-based chatbots can't handle natural language. Generic LLM integrations invent prices, availability, and policies — damaging trust and reputation.

The market needed an agent that speaks naturally, knows the actual catalog, books real appointments, and never invents information it doesn't have.

The Architecture: M1 → M2 → M3

Custom cognitive orchestration — no LangChain, no AutoGen.

Instead of relying on external orchestration frameworks, VioAgents uses a purpose-built three-moment system with strict data validation at each gate. This guarantees the agent never responds with invented data. 

M1 · Conversational Receptor
   └─ Understands user intent
   └─ Defines need_package: {readiness, missing_requirements}
   └─ Gates: if data is missing → asks user, never proceeds blindly

M2 · Technical Specialist
   └─ Executes tool calls (CSV catalog search, Google Calendar query)
   └─ Returns exact structured data (real prices, real time slots)
   └─ Cannot respond to user — only returns data to M3

M3 · Grounded Responder
   └─ Receives exact data from M2
   └─ Composes natural language response
   └─ Prohibited from inventing any price, product, or availability
   └─ If M2 returned empty → transparently says "not available"
The Three Agents
Sales Agent

Intelligent product search over CSV catalogs. Multi-query support, cross-sell suggestions, and PDF quotation generation triggered by providing the customer's name.

Booking Agent

Real Google Calendar integration. Checks live availability, blocks slots in real time, prevents duplicates, and suggests alternatives for taken time slots — in natural conversation.

FAQ / Knowledge Agent

Consumes structured Markdown knowledge bases. Smart human handoff: when frustration or escalation is detected, the bot pauses and notifies a human operator with full conversation context.

Multi-tenancy & Scalability

Onboard a new client in minutes, not days.

VioAgents uses a three-level configuration hierarchy: Domain → Niche → Business. The cognitive rules and sector vocabulary are inherited automatically — only business-specific policies and catalog files need to be added per client. Embedding the widget requires exactly 2 lines of code. 

Domain   Sales / Booking / FAQ  →  global behavior rules
Niche    Ferreterías / Clínicas →  sector vocabulary + categories
Business Ferretería López       →  tenant_defaults.md + CSV catalog

Technical Challenges & Solutions

AI Hallucinating Prices and Stock

Problem

Basic LLM integrations will often invent a price or confirm an appointment slot that doesn't exist just to sound helpful to the user.

Solution

The M3 architectural isolation. M3 cannot access internal weights for knowledge; it only formats the strict JSON return value from M2 into markdown tables.

Frustrating Bot Loops

Problem

Users hate when bots get stuck in loops and refuse to pass them to a human, leading to dropped leads and bad reviews.

Solution

Native sentiment analysis and explicit handoff detection. If frustration is detected, the bot pauses itself automatically and pages a human operator to take over the chat.
